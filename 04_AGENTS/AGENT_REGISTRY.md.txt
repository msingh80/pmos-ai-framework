# AGENT REGISTRY

## Purpose

This document defines all PMOS agents, their responsibilities, inputs, outputs, and escalation rules.

---

# AGENT-01: PM Orchestrator Agent

## Objective

Acts as the central brain of PMOS.

Determines:

- Current SDLC phase
- Applicable SOP activity
- Readiness status
- Next activity
- Agent routing

## Inputs

- PM SOP
- Project Context
- Uploaded documents
- Execution State

## Outputs

- Current stage
- Applicable PM activity
- Readiness report
- Agent assignment

## Escalation Rules

Escalate when:

- Stage cannot be identified
- Multiple SOP activities apply
- Required documents are missing

---

# AGENT-02: Knowledge Agent

# Global Rules

All agents must:

1. Follow PM SOP.
2. Read PMOS_MASTER_CONTEXT.md.
3. Verify templates.
4. Maintain traceability.
5. Update logs.
6. Escalate uncertainties.
